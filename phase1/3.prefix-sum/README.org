* Notes
- Prefix sum is fundamentally a "historical state" technique. You continuously record the past and ask whether a previous state can make the current range valid.
- Exact-count subarray problems are one of the strongest signals for prefix sum.
- The biggest conceptual leap is realizing:
  currentPrefix - target
  represents a required historical state.
- If negative numbers exist, sliding window should immediately become suspicious.
- Problems 523 and 974 are usually where modulo-based prefix normalization fully clicks.
- Problem 560 (Subarray Sum Equals K) is the foundational prefix hashmap problem for the entire topic.
- Problem 862 is important because it demonstrates that prefix sums alone are sometimes insufficient; structural optimization may also be required.
- Sliding window is the dynamic complement to static prefix accumulation.
