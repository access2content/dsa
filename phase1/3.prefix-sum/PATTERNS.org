* Notes
- Prefix sum patterns are usually hidden behind subarray language rather than explicit range-sum language.
- Exact-count subarray problems are one of the strongest signals for prefix sum.
- The biggest pattern leap is recognizing:
  currentState - targetState
  instead of trying to construct subarrays directly.
- If negative numbers exist, sliding window should immediately become suspicious.
- Problems 523 and 974 are usually where modulo-based prefix normalization fully clicks.
- Problem 862 is important because negatives destroy monotonic behavior, requiring structural optimization beyond plain prefix sums.
- Sliding window is the dynamic complement to static prefix accumulation.
