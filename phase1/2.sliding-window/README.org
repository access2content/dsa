* Core Idea
Instead of recalculating every subarray independently, maintain a contiguous window [left, right] and adjust it dynamically. Sliding window is fundamentally about maintaining a dynamically valid contiguous region while updating state incrementally.

The entire technique is about:
- Expanding when the window is valid/useful
- Shrinking when the window becomes invalid/excessive
- Reusing previous work instead of recomputing

The important realizations are:
- You avoid recomputing full ranges
- State evolves incrementally
- Boundaries encode logical validity
- Pointer movement represents proof that discarded states are permanently irrelevant

* How to solve
After every problem, write:
- Invariant — “What is always guaranteed to be correct?”
- Expansion rule — “When can the right pointer expand?”
- Shrink rule — “When must the left pointer move?”
- Why shrinking is safe — “Why can removed elements never help future windows?”
