* Mechanics and Questions
** Phase 1 — The Three Mechanics [0/3]
These problems establish the physical mechanics of sliding windows.

Core mechanics:
- Fixed Window — Maintain an exact window size while updating incrementally.
- Variable Window (Smallest) — Expand greedily, then shrink until invalid.
- Variable Window (Longest) — Expand continuously and shrink only when constraints break.

- [ ] [[https://leetcode.com/problems/maximum-average-subarray-i/description/][643. Maximum Average Subarray I]] (Fixed window fundamentals)
- [ ] [[https://leetcode.com/problems/minimum-size-subarray-sum/description/][209. Minimum Size Subarray Sum]] (Shrink until invalid)
- [ ] [[https://leetcode.com/problems/longest-substring-without-repeating-characters/description/][3. Longest Substring Without Repeating Characters]] (Expand until duplicate)
** Phase 2 — The “Aha!” Shifts [0/4]
These problems transform sliding window from a mechanical technique into a strategic reasoning tool. This phase is not about code structure, it is about cognitive breakthroughs.

Examples:
- invalid windows can still be useful
- counting can emerge from boundaries
- exact-K can be transformed

Core mechanics:
- K-Tolerance Windows — Allow temporary invalidity while preserving optimality.
- Frequency Matching — Compare dynamic frequency states efficiently.
- Window Counting — Count many valid subarrays mathematically using boundaries.
- Counting via Transformation — Convert harder counting problems into reusable counting templates.

- [ ] [[https://leetcode.com/problems/max-consecutive-ones-iii/description/][1004. Max Consecutive Ones III]] (K-tolerance windows)
- [ ] [[https://leetcode.com/problems/permutation-in-string/description/][567. Permutation in String]] (Frequency matching)
- [ ] [[https://leetcode.com/problems/subarray-product-less-than-k/description/][713. Subarray Product Less Than K]] (Sliding window counting)
- [ ] [[https://leetcode.com/problems/subarrays-with-k-different-integers/description/][992. Subarrays with K Different Integers]] (Exactly-K transformation)
** Phase 3 — Advanced Window Reasoning [0/4]
These problems teach advanced invariants, optimized data structures, and non-obvious problem transformations.

Core mechanics:
- Monotonic Window Optimization — Maintain monotonic candidates for O(1) max/min retrieval.
- Problem Inversion — Transform edge-removal problems into retained-middle windows.
- Multi-Constraint Shrinking — Maintain several validity constraints simultaneously.
- Structural Window Transformations — Recognize hidden sliding-window formulations.

- [ ] [[https://leetcode.com/problems/sliding-window-maximum/description/][239. Sliding Window Maximum]] (Monotonic deque)
- [ ] [[https://leetcode.com/problems/minimum-operations-to-reduce-x-to-zero/description/][1658. Minimum Operations to Reduce X to Zero]] (Problem inversion)

- [ ] [[https://leetcode.com/problems/binary-subarrays-with-sum/description/][930. Binary Subarrays With Sum]] (Counting transformation)

* How to solve
After every problem, write:
- Invariant - “What is always guaranteed to be correct?”
- Pointer movement rule - “What condition allows this pointer to move?”
- Why movement is safe - “Why can I permanently ignore the element(s) I just passed?”

Example:
Invariant:
- Array is sorted.

Movement rule:
- If sum too small → move left.
- If sum too large → move right.

Why safe:
- Sorted order guarantees discarded pairs
- cannot become valid later.
